# Docker Production Best Practices

Репозиторий демонстрирует применение лучших практик контейнеризации на примере бэкенда системы хранения энергии (ESS).

### Примененные паттерны:
1. **Multi-stage Builds (Многоэтапная сборка):** Разделение сборки зависимостей и финального запуска. Финальный образ не содержит компиляторов (`gcc`) и весит в 4 раза меньше.
2. **Запуск не от root (Non-root user):** Процесс внутри контейнера изолирован под пользователем `appuser`, что исключает риски компрометации хост-системы.
3. **Оптимизация кэширования слоев:** Тяжелые операции (`pip install`) вынесены выше копирования исходного кода (`COPY main.py .`), что ускоряет сборку при изменении логики приложения.
4. **Легковесный базовый образ:** Использование официального образа `python:slim` вместо тяжелого `ubuntu`.
