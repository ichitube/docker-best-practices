# Docker Production Best Practices

Этот репозиторий является шаблоном и демонстрирует применение лучших практик (Best Practices) при контейнеризации Python-приложений для Production-среды. 

### Реализованные архитектурные паттерны:

1. **Multi-stage Builds (Многоэтапная сборка):** 
   Разделение процесса на этап компиляции зависимостей и финальный запуск. В готовый образ (Stage 2) не попадают тяжелые системные утилиты (например, `gcc`), что радикально уменьшает размер итогового контейнера и снижает площадь атаки (attack surface).

2. **Безопасность (Non-root user):** 
   По умолчанию Docker запускает процессы от имени пользователя `root`. В данном шаблоне создан выделенный непривилегированный пользователь `appuser`, что является обязательным требованием безопасности в enterprise-инфраструктуре.

3. **Оптимизация кэширования слоев (Layer Caching):** 
   Инструкции в `Dockerfile` расположены от наименее изменяемых к наиболее изменяемым. Тяжелая установка библиотек (`pip install`) вынесена наверх, а копирование исходного кода (`COPY main.py .`) — вниз. Это позволяет пересобирать образ за секунды при изменении бизнес-логики.

4. **Использование легковесных базовых образов:** 
   Применяется официальный образ `python:slim` на базе урезанного Debian, что обеспечивает баланс между минимальным размером и отсутствием проблем с C-библиотеками (в отличие от `alpine`).
